# SWOT战略分析技能

> 本技能是**交互引导式**工具：Agent 扮演引导者/采访者，通过强制交互向用户逐项索取真实信息，而非由 Agent 凭经验代写。四方格内容必须由用户提供，战略组合方可沿用本技能逻辑自动推导。

## 一、能力说明（调用前置）

**本技能能做什么**
- 引导用户对「部门 / 产品 / 项目 / 供应商」进行系统的 SWOT 分析
- 通过强制交互，逐项采集 S（优势）、W（劣势）、O（机会）、T（威胁）的真实信息
- 基于用户提供的信息，沿用四战略组合逻辑（SO / WO / ST / WT）生成战略建议
- 输出双版本成果：结构化 `MD 文档` + 精美 `网页版 HTML`（用于演示）

**本技能不能做什么（边界）**
- ❌ 不替用户撰写 S/W/O/T 任何象限的内容
- ❌ 不替用户判定某条信息应归为「机会」还是「威胁」（可给归类建议，决定权在用户）
- ❌ 不替用户设定 P1/P2/P3 优先级（可给方法论建议）
- ❌ 不对用户未提供的象限凭空编造（缺失须标注「供参考·待确认」并提示失真）
- ❌ 不保证分析结果正确——分析质量取决于用户提供信息的真实与充分程度
- ❌ 非用户主动提供质量战略时，不做质量管理专项校验（本工具通用，质量是应用方向之一）

**开始前的输入要求（缺失则主动提示）**
- 分析对象是什么（部门 / 产品 / 项目 / 供应商，及具体名称）
- 分析范围（时间范围、业务范围、特定场景）
- 用户特别关注点（如成本控制、市场拓展、质量提升）
- 若用户希望战略建议服务于某上层战略/目标，请提供该战略文本（**可选**，不提供则标注「待提供战略后复核」，不卡流程）

## 二、核心机制：四象限强制交互锁

SWOT 分析拆为 **4 道必须依次通过的采集门**。每道门未获得用户**实质输入**前，不得推进到下一门或战略生成阶段。

| 门 | 象限 | Agent 在本门的行为 | 锁规则 |
|----|------|-------------------|--------|
| 门1 | 优势 S | 给出思考维度 + 示例 → **强制提问索要**用户的具体情况 | 无实质输入 → 提示缺失，停在门1 |
| 门2 | 劣势 W | 同上 | 同上 |
| 门3 | 机会 O | 同上 | 同上 |
| 门4 | 威胁 T | 同上 | 同上 |

**交互原则（贯穿四门）**
- ✅ Agent 可以：澄清模糊表述、提供示例引导、追问细节、提示"这条更像 O 还是 T"（给建议，不定论）
- ❌ Agent 不可：替用户写内容、替用户归类、替用户决断
- 每门引导话术与示例见 `references/interaction_guide.md`，四对象思考维度见 `references/thinking_dimensions.md`；真实对话示范见 `references/dialogue_examples.md`

**跳过与失真提示**
- 若用户确无某门信息，允许临时跳过，但该象限标注「供参考·待确认」，**并明确告知：缺失将导致对应战略组合（如 WO/WT）失真或无法生成**
- 演示场景下用户可用占位内容临时填充，但须标注「演示占位」

## 三、战略组合（沿用本技能逻辑）

四门收齐后，按 `references/strategy_logic.md` 的四战略组合逻辑自动推导：
- **SO 战略（进攻型）**：用优势抓机会
- **WO 战略（改善型）**：用机会补劣势
- **ST 战略（防御型）**：用优势抗威胁
- **WT 战略（生存型）**：减劣势避威胁

每条战略建议须标注「基于您提供的 XX 要素」，交由用户确认/调整。优先级（P1/P2/P3）由用户拍板，Agent 仅给方法论建议。

## 四、输出：双版本（演示用）

收集完成 + 战略生成后，**先向用户展示报告大纲并确认**（见 `assets/SWOT分析_输出大纲.md`），确认后再生成：

1. `SWOT分析_报告.md` — 标准结构化文档
2. `SWOT分析_报告.html` — 精美网页版（四象限卡片 + 战略卡片 + 响应式 CSS），用于演示

生成方式：将结构化结果（JSON）交给 `scripts/build_report.py`，自动产出 MD 与 HTML 两版；HTML 套用 `assets/swot_report_template.html` 模板。

## 五、标准工作流程

1. **开场说明**：介绍本技能能力与边界、所需输入（第一节），缺失则主动提示
2. **门1 采集 S**：引导话术 + 强制提问 → 用户填 → 未填不进
3. **门2 采集 W**：同上
4. **门3 采集 O**：同上
5. **门4 采集 T**：同上
6. **战略生成**：按四组合逻辑推导，每条标注来源要素，交用户确认
7. **大纲确认**：展示报告大纲，用户确认结构
8. **双版本输出**：生成 MD + 精美 HTML

## 六、质量规范（本技能开发通用强制规则）

1. **调用前置说明**：每次调用先简介能力/边界/输入要求（见第一节），缺失信息主动提示
2. **输入载体判断**：若用户提供现有材料（.docx/.pdf/.xlsx/.md），调用文档处理技能（docx / markitdown / pdf）解析为文本辅助采集，不替代用户口述
3. **缺失信息标注**：任一象限缺失，主动说明对结果的影响，标注「供参考·待确认 / 演示占位」
4. **大纲先确认**：生成长文档前，必先展示大纲由用户确认结构
5. **边界清晰**：能做什么、不能做什么（见第一节）须显式声明

## 八、相关技能联动（战略分析工具链）

SWOT 是工具链的**收敛中枢**：把外部分析与内部评估汇聚成战略组合。本链由 6 个交互引导式技能串联，遵循"外部扫描 → 内部评估 → 收敛战略 → 落成目标 → 分解落地"：

`pestel-analysis`（宏观外部）→ `porters-five-forces`（行业外部）→ `vrio-analysis`（内部）→ **`swot-analysis`（收敛）** → `smart-goal-setter`（落成目标）→ `qms-objective-decomposition`（质量分解落地）

- **上游输入（直接引用其结论，不重复采集）**：
  - `pestel-analysis`（宏观环境）→ 提供 O/T（机会/威胁）的宏观来源
  - `porters-five-forces`（行业结构）→ 提供 O/T 的行业竞争来源
  - `vrio-analysis`（内部资源能力）→ 提供 S/W（优势/劣势），并标识可成"持续竞争优势"的资源
- **下游输出**：SO/WO/ST/WT 战略建议 → 交给 `smart-goal-setter` 落成可衡量的 SMART 目标
- **质量场景延伸**：若战略方向落在质量领域，经 `smart-goal-setter` 优化后，交由 `qms-objective-decomposition` 做质量目标分解与审核
- **通用性**：这 6 个工具均不锁质量管理领域，质量只是应用方向之一；均通过"强制交互锁"保证数据真实、"双版本输出"便于演示。
