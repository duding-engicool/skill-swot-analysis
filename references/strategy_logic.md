# 四战略组合生成逻辑（沿用原技能）

四门采集完成后，Agent 基于用户提供的 S/W/O/T 要素，按以下逻辑推导战略组合。**每条战略建议须标注其来源要素（如"基于 S2+O1"），交由用户确认。**

---

## SO 战略（优势 × 机会，进攻型）

- 思路：用内部优势抓住外部机会，实现突破
- 生成规则：取最强的 2-3 个优势 与 最大的 2-3 个机会 进行配对组合
- 输出：每条建议 = "利用[优势X]，抓住[机会Y]，实现[具体动作]"
- 例：S=检测设备齐全 + O=客户质量要求提升 → 推出高端检测增值服务

## WO 战略（劣势 × 机会，改善型）

- 思路：借外部机会弥补或克服内部劣势
- 生成规则：识别关键劣势与可借力的机会，设计改进路径
- 输出：每条建议 = "借[机会Y]契机，补齐[劣势X]"
- 例：W=工具应用少 + O=精益管理导入 → 借导入期培训 SPC/APQP
- ⚠️ 若 W 象限缺失，WO 战略无法生成或失真，须标注

## ST 战略（优势 × 威胁，防御型）

- 思路：用优势应对或规避外部威胁
- 生成规则：用优势资源化解威胁，或建立防御壁垒
- 输出：每条建议 = "以[优势X]抵御[威胁Z]"
- 例：S=客户认可度高 + T=竞品技术领先 → 强化客户关系壁垒

## WT 战略（劣势 × 威胁，生存型）

- 思路：减少劣势、规避威胁，保基本运转
- 生成规则：识别最危险的"劣势+威胁"组合，制定紧急应对
- 输出：每条建议 = "降低[劣势X]暴露、规避[威胁Z]"
- ⚠️ 若 W 或 T 缺失，WT 战略无法生成或失真，须标注

---

## 优先级建议（方法论，不由 Agent 定档）

- 进攻型（SO）通常优先，但需结合用户资源与紧迫度
- 生存型（WT）在威胁迫近时升为 P1
- Agent 给出"建议优先级及理由"，最终 P1/P2/P3 由用户拍板

## 输出格式

```
SO战略（进攻型）：
• 建议1（具体可执行，基于 Sx+Oy）
• 建议2

WO战略（改善型）：
• 建议1（基于 Wx+Oy）

ST战略（防御型）：
• 建议1（基于 Sx+Tz）

WT战略（生存型）：
• 建议1（基于 Wx+Tz）

重点行动方向（2-3条，建议优先级 P1/P2/P3 + 理由）：
• P1：[行动] — 理由
• P2：[行动] — 理由
• P3：[行动] — 理由
```
